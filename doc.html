Documentación Técnica

House Pricing Automation API es una solución desacoplada diseñada para procesar masivamente tasaciones inmobiliarias. Separa la interfaz de usuario (PHP) del procesamiento pesado (Python/Selenium) para evitar timeouts y problemas de concurrencia.

El sistema permite subir un Excel con roles y comunas, y devuelve un reporte enriquecido con datos de "Informe de Antecedentes" (PDF OCR) y comparables de mercado (Web Scraping).

2. Arquitectura del Sistema

El sistema sigue un patrón Cliente-Servidor Asíncrono con gestión de estado en memoria.

+----------------+ +--------------------+ +-------------------------+ | CLIENTE (PHP) | ---> | API (FastAPI) | ---> | WORKER (Lógica) | | (Interfaz UI) | <--- | (Gestor de Cola) | <--- | (Selenium/PDF/Pandas) | +----------------+ +--------------------+ +-------------------------+ | | | 1. Sube Archivo 2. Genera UUID Task 3. Ejecuta Paso 0 4. Polling Estado <------ 5. Lee Estado Memoria 6. Callbacks Progreso 7. Descarga Final 8. Sirve Archivo 9. Genera Excel
  • PHP (Frontend): Solo maneja la UI y peticiones HTTP. No procesa datos.
  • FastAPI (Backend): Orquestador. Recibe archivos y lanza hilos en BackgroundTasks.
  • Pipeline (Core): Scripts de Python puros que ejecutan la lógica de negocio "sagrada".

4. Requisitos del Servidor (Linux)

El sistema está optimizado para Ubuntu 20.04+ o Fedora 38+.

Componentes Base

  • Python 3.10+
  • Google Chrome Stable: Necesario para Selenium.
  • Permisos: El usuario que ejecute el script debe poder crear carpetas.
Nota sobre Drivers: No es necesario descargar chromedriver manualmente. Selenium 4.10+ usa Selenium Manager para descargarlo automáticamente según la versión de Chrome instalada.

5. Guía de Instalación

Paso 1: Instalar Google Chrome

# Ubuntu / Debian
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt install ./google-chrome-stable_current_amd64.deb

# Fedora / CentOS
sudo dnf install google-chrome-stable

Paso 2: Configurar Entorno Python

# Clonar o copiar archivos
cd /ruta/a/tu/proyecto

# Crear entorno virtual (Recomendado)
python3 -m venv venv
source venv/bin/activate

# Instalar dependencias
pip install -r requirements.txt

Paso 3: Ejecutar la API

# Ejecución directa (Desarrollo)
python server.py

# Ejecución Producción (Segundo plano)
nohup python server.py > /dev/null 2>&1 &

La API iniciará en el puerto 3000.

6. Pipeline de Lógica (Core)

El núcleo del sistema reside en la carpeta api/. La lógica original se mantiene intacta, pero se le han inyectado callbacks para reportar progreso.

  • Paso 0 (Descargas): Usa Selenium para descargar PDFs desde el portal de antecedentes.
    Clave: Manejo robusto de reintentos al mover archivos descargados.
  • Paso 1 (PDF Parsing): Usa pdfplumber. Extrae roles, avaluos y coordenadas espaciales de los hipervínculos del PDF.
  • Paso 2 (Enriquecimiento): La etapa más pesada. Usa ThreadPoolExecutor y Selenium para buscar comparables en HousePricing.
    Importante: Implementa un contador de progreso thread-safe.
  • Paso 3 (Reporte): Cruza toda la data (JSON) y genera un Excel con 5 hojas relacionales. Usa openpyxl para formateo.

7. Referencia de API

Endpoints disponibles en http://TU_IP:3000.

Método Endpoint Descripción
GET /health Verifica que el sistema esté operativo.
POST /upload-process Sube archivo Input. Retorna {"task_id": "uuid"}.
Param: file (Binary)
GET /status/{task_id} Polling de estado. Retorna progress (0-100) y status.
POST /cancel/{task_id} Envía señal de interrupción (Event Set) al proceso.
GET /download/{task_id} Descarga el Excel final (Solo si status == "completed").

8. Integración Frontend (PHP)

El archivo index.php contiene la lógica de cliente. Como la arquitectura separa el servidor Web del servidor de Procesamiento, se debe configurar la IP manualmente.

Configuración Crítica

Editar la línea dentro del bloque <script> en index.php:

// index.php

// REEMPLAZAR con la IP real donde corre server.py
const API_URL = "http://192.168.1.50:3000"; 

// NO usar 'localhost' si el PHP y Python están en servidores distintos
// o si se accede desde otra máquina de la red.
